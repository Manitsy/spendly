❯ what does this project do                                                     

  Read 2 files (ctrl+o to expand)

⏺ This is a Flask-based expense tracker app called Spendly — structured as a
  learning project where students build it step-by-step.

  Current state (scaffold/starter):
  - Landing page, login, and register routes are wired up with templates
  - Placeholder routes exist for: logout, profile, add/edit/delete expenses

  Planned full functionality (to be implemented by students):
  - Auth — user registration, login, logout (Steps 3–4)
  - SQLite database — get_db(), init_db(), seed_db() (Step 1)
  - Expense CRUD — add, edit, delete expenses (Steps 7–9)

  It's a guided tutorial project for learning Flask + SQLite, not a finished
  application.

❯ what tech stack does this project use                                         

⏺ Based on the code and files in the project:

  Backend
  - Python — primary language
  - Flask 3.1.3 — web framework
  - SQLite — database (via Python's built-in sqlite3)
  - Werkzeug 3.1.6 — WSGI utilities (password hashing, routing)

  Frontend
  - HTML/Jinja2 — templating (templates/)
  - CSS — custom styles (static/css/style.css)
  - JavaScript — client-side logic (static/js/main.js)

  Testing
  - pytest 8.3.5 — test framework
  - pytest-flask 1.3.0 — Flask test integration

  No external database, no frontend framework, no ORM — intentionally minimal
  for a learning project.

  - app.py imports from database/db.py (once implemented) for DB access
